## ðŸŒ Deployment (Cloudflare Pages)

This project is configured for deployment on Cloudflare Pages using the Edge Runtime.

### Build Command
The custom build command handles the specifics of the Cloudflare adapter:

```bash
npm run pages:build
```

This command:
1.  Builds the Next.js app using `@cloudflare/next-on-pages`.
2.  Generates the `_worker.js` output.
3.  Creates an `.assetsignore` file to prevent upload errors.

### Configuration
-   **Output Directory**: `.vercel/output/static`
-   **Compatibility Flags**: `nodejs_compat`
-   See `wrangler.json` for detailed configuration.

## ðŸ¤– AI Studio Workflow

This project utilizes a structured AI-driven development workflow located in the `ai-studio/` directory.

### Key Files in `ai-studio/`

1.  **`lead-web-architect-creative-director-system-instructions.md`**
    *   **Purpose**: These are the **System Instructions** used to configure the lead AI model in [Google AI Studio](https://aistudio.google.com/).
    *   **Role**: Defines the persona (Lead Web Architect & Creative Director), design mandate ("Tech Dark" aesthetic), and strict output guardrails. This file ensures the AI "architect" understands the vision before writing a single line of code.

2.  **`website-upgrade-plan.md`**
    *   **Purpose**: This is the **Implementation Plan** generated by the AI Studio model based on the system instructions.
    *   **Recommendation**: Just select "Next.js" preset and let Cloudflare detect the settings. It usually defaults to `npm run build` and output `.next`.
